Introdução

O Git é um sistema de controle de versão popular. Foi criado por Linus Torvalds em 2005, e tem sido mantido por Junio Hamano desde então.

É usado para:

  • Alterações de código de acompanhamento
  • Rastreando quem fez alterações
  • Colaboração de codificação

O que o Git faz?

  • Gerenciar projetos com repositórios
  • Clonar um projeto para trabalhar em uma cópia local
  • Controle e controle as alterações com a preparação e a confirmação
  • Ramificação e mesclagem para permitir o trabalho em diferentes partes e versões de um projeto
  • Efetuar pull da versão mais recente do projeto para uma cópia local
  • Enviar atualizações locais para o projeto principal

Instalação do Git

Você pode baixar o Git gratuitamente no seguinte site: https://www.git-scm.com/

verificando se o GIT está instalado

        git --version

Configurar o Git

Agora deixe o Git saber quem você é. Isso é importante para sistemas de controle de versão, como cada commit do Git usa essas informações:

        git config --global user.name "test"
        git config --global user.email "test@email.com"

Altere o nome de usuário e o endereço de e-mail para o seu. Você provavelmente também desejará usar isso ao se registrar no GitHub mais tarde.

Use global para definir o nome de usuário e o e-mail para cada repositório em seu computador. Se você quiser definir o nome de usuário/e-mail apenas para o repositório atual, poderá remover global.

Incializando o GIT

você pode inicializar o Git

        git init

Verificar estatus do repositório

        git status

Os arquivos na pasta do repositório Git podem estar em um dos 2 estados:

  • Rastreados - arquivos que o Git conhece e são adicionados ao repositório
  • Não rastreados - arquivos que estão em seu diretório de trabalho, mas não adicionados ao diretório repositório

Quando você adiciona arquivos a um repositório vazio pela primeira vez, todos eles são não rastreado. Para que o Git os rastreie, você precisa prepará-los ou adicioná-los ao ambiente de preparo.
