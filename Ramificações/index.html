Trabalhando com ramificações do Git

No Git, um branch é uma versão nova/separada do repositório principal.

As ramificações permitem que você trabalhe em diferentes partes de um projeto sem afetar o ramo principal.
Quando o trabalho estiver concluído, uma ramificação poderá ser mesclada com o projeto principal.
Você pode até alternar entre ramificações e trabalhar em diferentes projetos sem eles interferindo uns com os outros. 

        git branch [name-branch]

checkout é o comando usado para fazer check-out de um arquivo . Movendo-nos do atual , para o especificado no final do comando

Agora movemos nosso espaço de trabalho atual do branch master para o novo branch.

        git checkout

Agora imagine que ainda não terminamos com novo branch, mas precisamos corrigir um erro no master.
Eu não quero mexer com o mestre diretamente, e eu não quero mexer com hello-world-images, uma vez que ainda não está pronto.
Por isso, criamos uma nova filial para lidar com a emergência: 

        git checkout -b [name-new-branch]

Agora criamos um novo branch do master e mudamos para ele. Nós podemos Corrija o erro com segurança sem perturbar as outras ramificações.

Mesclar ramificações

Primeiro, precisamos mudar para o branch master, e mesclamos o branch atual (master) com emergency-fix

        git checkout master
        git marge [name-branch]

Como o novo branch veio diretamente do master e nenhuma outra alteração foi feita no master enquanto estávamos trabalhando, o Git vê isso como uma continuação do master. Portanto, ele pode "avançar rapidamente", apenas apontando master e o novo branch para o mesmo commit.

Como master e o novo branch são essencialmente os mesmos agora, podemos excluir emergency-fix, pois não é mais necessário

        git branch -d [name-branch]
