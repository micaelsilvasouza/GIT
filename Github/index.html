GitHub

Acesse o https://www.github.com/ e inscreva-se em uma conta, lembre-se de usar o mesmo endereço de email usado na configuração do Git.

Agora que você criou uma conta do GitHub, entre e crie um novo repositório.

Enviar o repositório local para o GitHub

Como já configuramos um repositório Git local, vamos para push o GitHub.

        git remote add origin https://github.com/w3schools-test/hello-world.git

Copie o URL ou clique na área de transferência.

Especifica que você está adicionando um repositório remoto, com o especificado , como um para o repositório Git local.

Agora vamos enviar nosso branch master para a url de origem e defini-lo como o branch remoto padrão.

        git push --set-upstream origin master

Puxando para se manter atualizado com as alterações

Sempre que você começar a trabalhar em um projeto, deverá obter o mais recente alterações em sua cópia local.

O pull é usado para extrair todas as alterações de um repositório remoto no branch em que você está trabalhando.

        git pull origin

É assim que você mantém seu Git local atualizado a partir de um repositório remoto.

Agora envie nossas alterações para nossa origem remota:

        git push origin

Ramificações

Lá no GitHub crie um novo Branch. Digite um nome descritivo e clique em Criar ramificação. O agora deve ser criado e ativo. Você pode confirmar qual ramificação você estão trabalhando olhando para o botão de ramificação

Agora continue trabalhando em nosso novo em nosso Git local. Vamos do nosso repositório GitHub novamente para que nosso código esteja atualizado

        git pull

Agora podemos usar a opção de ver todas as filiais locais e remotas

        git branch -a

branch -r é apenas para filiais remotas.

Vemos que o branch está disponível remotamente, mas não em nosso git local. Vamos conferir

        git branch [name-branch]

É assim que você puxa um branch do GitHub para o Git local.

Para enviar um branch local para o github

        git push origin [name-branch]
